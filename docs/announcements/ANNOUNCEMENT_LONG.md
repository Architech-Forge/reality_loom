# Reality Loom: Opening the Foundation for Reality-Oriented Computing

I’m releasing the early public foundation for **Reality Loom**.

Reality Loom is my foundation for **Reality-Oriented Computing** — a new way to build software as living worlds instead of screens wrapped around data.

For decades, most software has been built around pages, dashboards, forms, feeds, and components.

That model is useful, but it is not enough for the next generation of intelligent software.

AI-native products need more than prompts and chat boxes. Spatial interfaces need more than canvases. Personalized systems need more than user profiles. Enterprise systems need more than dashboards.

They need a model of what is true.

They need a runtime that understands possible state versus committed state.

They need interfaces projected from meaning, not assembled from disconnected UI pieces.

That is what Reality Loom is for.

```text
Describe reality.
Compute understanding.
Project experience.
```

## What Reality Loom Introduces

Reality Loom is the open foundation for:

- **Reality-Oriented Computing** — the philosophy and category
- **World Graph Engine** — the graph substrate for living worlds
- **World Interaction Language** — the protocol for actors and systems
- **World Definition Language** — the declarative language for defining worlds
- **World Intermediate Representation** — the compiler-internal representation
- **Spatial Living Interface** — the projection layer for living experiences

At the center is a shift in how software is modeled:

```text
Page        → World
Component   → Entity / Projection
State       → Snapshot / Diff / Trace
User action → WIL Message
AI output   → Proposal / Candidate / Explanation
UI          → Projection from world truth
```

## Why Open Source This

I’m open-sourcing the foundation because I think this deserves to exist as a public transition point for a new UX paradigm.

The private products built on top remain private.

Reality Loom is the foundation.

The applications are the proof.

## Public vs Private

This repository is for the public foundation:

- Codex volumes
- architecture
- WGE/WIL/WDL/SLI specifications
- reference types
- deterministic fixtures
- public diagrams and examples

It does not include proprietary products, private prompts, private app code, commercial runtime logic, customer data, private traces, or API keys.

## Ownership

Reality Loom is currently owned and published by **Architech Forge**.

The long-term intent is for Reality Loom to become its own dedicated company/platform steward — the long-term home for the category, standards, and ecosystem.

## Current Status

This is an early public foundation.

The Codex is already substantially built out, but the project should be treated as draft-stage / pre-1.0.

The goal of this release is not to declare the final answer.

The goal is to make the direction public:

```text
Software should become world-aware.
AI should operate inside traceable reality.
Interfaces should emerge from meaning.
```

## The Larger Bet

The future of software will not be a better dashboard.

It will be systems that understand context, constraints, objectives, relationships, history, privacy, and possibility.

Reality Loom is my attempt to define that foundation in public.
