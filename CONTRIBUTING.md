# Contributing

Reality Loom is currently in early public foundation stage.

Contributions are welcome, but the project is still defining its governance model.

## Contribution Principles

Contributions should preserve:

- determinism
- traceability
- candidate vs reality separation
- authority and permission boundaries
- renderer agnosticism
- AI boundedness
- privacy-aware design
- stable public protocol language

## Good Contributions

Good early contributions include:

- clarifying docs
- deterministic test fixtures
- schema improvements
- typo fixes
- architecture examples
- WIL/WDL examples
- compliance tests
- reference implementation notes

## Avoid

Please do not contribute:

- proprietary application code
- private prompts
- secrets
- user data
- vendor-locked assumptions
- code that bypasses runtime authority
- UI patterns that turn Reality Loom into a dashboard framework
